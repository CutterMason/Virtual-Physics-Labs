Virtual Physics 1 Labs 
Installation/To Run: 
IMPORTANT NOTE: DO NOT MESS WITH ANY FILE LOCATIONS, AS FILES ARE 
DEPENDENT ON OTHERS IN LOCATION 
Windows:  
Decompress “Project Builds”.  
Open the folder “Windows”. 
Run file called “Virtual Physics Labs 
Linux: 
Decompress “Project Builds” 
Open the folder “Linux” in terminal 
Run the command “chmod +x Linux.x86_64” 
Run the command “./Linux.x86_64” 
Mac: 
Unknown how to run as we did not get access to a Mac to try, but files are present! 
GitHub and Unity: 
Download Project from Github: https://github.com/CutterMason/Virtual-Physics-Labs  
Clone the repository to a local directory from GitHub. Then, inside Unity Hub, go to 1 
‘Projects’, click ‘New project’ and select local folder cloned from the repository.  
Open Project in Unity Version 6000.0.46f1 
Login Page 
New Account: 
Enter an email ending in @buffs.wtamu.edu for students, or @wtamu.edu for teachers.  
No other email will be accepted. 
Enter a password, and make sure to remember the password as no recovery is available.  
Select either student or teacher role. Roles correspond with the email entered. 
Returning Account: 
Enter the used email, associated password, and correct role to log into an already existing 
account. 
Main Menu 
Click “Tutorial” for a brief tutorial on the buttons that appear in the various labs. 
Preset Labs takes you to the selectable preset labs. Each lab has the necessary tools to complete 
the corresponding labs for the Physics I course. Currently, all labs are implemented. 
Click the “Saved Labs” button in the top right corner of the Preset Labs screen to access any 
saved labs currently on the account. “Published Labs” does similar to saved labs but with 
published labs. 
Click “Sandbox” to access the sandbox where custom labs can be built. The red arrow on the 
right of the screen opens a menu of spawnable items that can be used. 
Click “Grades” to access the grade menu to see any grades available. 
Click “Settings” to change the volume of the music and camera speed. The “Master Volume” and 
“Music Volume” sliders do the same thing, so use whichever. 
Preset Labs: 
Most objects will not move until clicking the “Play” button. 
If UI looks messed up in the preset labs/sandbox (i.e. small ui, ui not in corner, etc), please 
follow instructions to get the project files from GitHub and run through Unity. 
Additional Controls 
All additional controls have been detailed in the application in Settings/Controls.
